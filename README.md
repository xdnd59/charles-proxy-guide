## Charles 是什么？一句话说清楚

**Charles 是一款跨平台的网络代理抓包工具**，支持 macOS、Windows 和 Linux。它通过将自己设置为系统的网络代理服务器，拦截并记录设备发出的所有 HTTP/HTTPS 流量，让你能实时查看请求报文和响应内容，是开发调试、接口测试、弱网模拟的核心工具之一。

## Charles 能干什么？核心功能一览

很多人刚装上 Charles，只用来抓个接口看看数据，其实它的功能远不止于此。

**抓包与分析**，这是最基础的用途。所有经过 Charles 代理的请求，都会被完整记录下来，包括请求头、请求体、响应状态、响应内容，一目了然。Structure 视图按域名归类，Sequence 视图按时间排序，两种模式随时切换。

**SSL 解密**，这才是 Charles 的核心竞争力。默认情况下 HTTPS 请求是加密的，什么也看不到。配置好 SSL 代理并安装根证书之后，Charles 会把 HTTPS 内容明文展示给你，再也不用对着一堆 `<unknown>` 干瞪眼。

**弱网模拟**，做移动端测试的人应该很熟悉这个需求。Charles 可以调节带宽和延迟，模拟 2G、3G 等慢速网络，在实验室里复现用户在网络差的情况下遭遇的问题。

**断点与请求篡改**，设置断点后，Charles 会在请求发出或响应返回时暂停，让你手动修改参数，再放行。这在测试边界条件、模拟异常响应的场景下非常实用。

**重复测试**，对着某个请求，修改参数，重新发一遍。不用反复操作 App，直接在 Charles 里改了就发。

## 如何抓取 HTTPS 请求？SSL 证书配置详解

这是最多人卡住的地方。按顺序来：

1. 打开 Charles，点击菜单 `Help → SSL Proxying → Install Charles Root Certificate`，将证书添加到本机的钥匙串或证书存储中。
2. 找到刚安装的证书，右键「显示简介」，把信任策略改为「始终信任」。
3. 打开 `Proxy → SSL Proxying Settings`，勾选 `Enable SSL Proxying`，点 Add，Host 填 `*`，Port 填 `443`，保存。

这样，HTTPS 请求就能被 Charles 解密并明文显示了。

## 手机抓包怎么设置？iOS 和 Android 都说

这部分的核心前提是：**手机和电脑必须在同一个 Wi-Fi 下**。

### iOS 手机抓包

1. 在电脑 Charles 中查看本机 IP：`Help → Local IP Addresses`，记下电脑的 IP 地址。
2. iPhone 打开「设置 → 无线局域网」，点击当前 Wi-Fi 名右边的详情键，拉到最底部「HTTP 代理」，切换为「手动」，填入电脑 IP 和端口 `8888`。
3. 电脑上 Charles 会弹出提示，确认允许该设备连接。
4. 在 iPhone 的 Safari 浏览器里访问 `chls.pro/ssl`，下载并安装 Charles 根证书。
5. 打开「设置 → 通用 → 关于本机 → 证书信任设置」，找到 Charles 证书，打开信任开关。

完成以上步骤，iPhone 上所有 HTTP/HTTPS 请求就能被 Charles 捕获了。

### Android 手机抓包

Android 的配置步骤和 iOS 基本相同——连同一 Wi-Fi，手动配置代理为电脑 IP 加端口 8888，然后在浏览器中下载证书安装。

需要注意的是，Android 7.0（Nougat）之后系统对用户安装的证书信任级别有所限制，某些 App 会忽略用户证书而只信任系统级证书，这时需要借助 OpenSSL 将证书转换并推入系统证书目录，过程稍复杂一些，需要 root 权限。

## Charles 常用功能速查：这几个场景用得最多

**过滤域名**，避免请求太多看不过来。可以在 `Proxy → Recording Settings → Include` 里添加你关心的域名和端口，Charles 只记录匹配的请求。或者直接右键某个域名，点「Focus」，它就会置顶显示。

**重发请求**，右键某条请求，选「Repeat」，立刻重发一次。反复测试某个接口时非常省时。

**Map Local / Map Remote**，本地映射和远程重定向是高级调试功能。比如把线上某个 JS 文件映射到本地版本，你在本地修改代码，App 拿到的却是你的本地文件，方便线上问题的本地复现与调试。

**模拟弱网**，工具栏上有个小乌龟图标，点开就是网速限制设置，预设了多种移动网络的模拟场景，直接用就行。

## 常见问题 FAQ

**Q：Charles 提示「Failed to find Java VM」怎么处理？**

Charles 依赖 Java 运行环境。需要先从 Oracle 官网下载安装 JRE，注意 Charles 32 位版本需要 32 位 JRE，64 位版本对应 64 位 JRE，版本必须匹配。

**Q：Charles 抓不到 localhost 的请求，怎么回事？**

这是 Charles 的已知问题，无法直接抓取 localhost 域名的流量。解决方法是把请求地址换成 `localhost.charlesproxy.com`，Charles 将其映射到 127.0.0.1，就可以正常抓包了。

**Q：开启 Charles 后，之前配置的 VPN 或代理失效了，怎么解决？**

Charles 开启后会接管系统代理设置，导致其他代理工具失效。操作顺序很重要：**先启动 Charles，再连接 VPN**。如果关闭了 Charles，需要断开 VPN 重新连接。

**Q：手机抓包时，HTTPS 请求还是显示乱码/unknown，怎么处理？**

检查以下几个点：手机证书是否已安装并已在"证书信任设置"中设为完全信任；SSL Proxying Settings 中是否已添加 `*:443` 规则并勾选 Enable SSL Proxying；iPhone 建议用 Safari 而非第三方浏览器下载证书（部分浏览器下载的证书格式不对）。

**Q：在数码荔枝购买的激活码，能激活从官网下载的 Charles 吗？**

可以。数码荔枝销售的是 XK72 的正版授权，与官网下载的 Charles 完全兼容，激活码在官网购买页面的 Promo Code 处输入即可获得正版授权。

**Q：Charles 支持几台设备同时使用？**
